# Markdown File Schema for RAP Manager

All markdown files generated by the AI IDE should include YAML frontmatter at the top of the file.

## Frontmatter Fields

| Field | Required | Values | Description |
|---|---|---|---|
| `type` | Yes | `project-scope`, `user-journey`, `mental-snapshot` | Which tab this file belongs to |
| `subtype` | Yes | See below | Specific role of the document |
| `title` | Yes | Free text | Display title for the card |
| `journey` | Only for `user-journey` | Slug (e.g. `login-flow`) | Groups related journey files together |
| `createdAt` | Yes | ISO 8601 date string | When the file was created |

## Subtype Values by Type

### `project-scope`
- `initial-idea` — The initial project idea
- `competitor-features` — Closest competitor feature analysis
- `mvp-wbs` — Work breakdown structure of the MVP

### `user-journey`
- `journey-idea` — Initial idea for a user journey
- `journey-wbs` — Fleshed out WBS of the journey
- `journey-journal` — Journal/log of implementing the journey WBS

### `mental-snapshot`
- `snapshot` — Context capture for resuming work later

## Example Files

### Project Scope — Initial Idea
```markdown
---
type: project-scope
subtype: initial-idea
title: "RAP Manager Initial Idea"
createdAt: "2026-02-07T20:00:00Z"
---

# RAP Manager

A tool for managing development documentation...
```

### User Journey
```markdown
---
type: user-journey
subtype: journey-idea
title: "Login Flow"
journey: login-flow
createdAt: "2026-02-07T20:00:00Z"
---

# Login Flow Journey

Users should be able to log in with...
```

### Mental Snapshot
```markdown
---
type: mental-snapshot
subtype: snapshot
title: "Auth integration progress"
createdAt: "2026-02-07T20:00:00Z"
---

# Where I Left Off

Finished the login form component...
```

## File Location

Place all markdown files in the `DevDocs/{ProjectName}/` directory relative to the project root.
